Module std::iter

Ranges and iterators.

Iterators provide an abstraction for types that can be used to access their elements one-by-one, for example in for loop. The most common examples are ranges and collections. 

for i in 0..10 {
    println!("{}", i);
}

for i in &["a", "b", "c"] {
    println!("{}", i);
}

There is distinction between an iterable type and an iterator. Iterators are consumable objects that can only be used to iterate once. Iterable types are ones that can produce an iterator when needed.

Iterators can also be combined to create more complex types of iterators. See IteratorExt protocol for a reference of methods that can be used to combine iterators.

Functions

  • fn empty<T>() -> EmptyIterator<T>

    source

    Returns an iterator that produces no elements.

    Example

    let range = std::iter::empty::<()>();

assert_eq!(range.next(), Option::none());

  • fn from_fn<F, T>(func: F) -> FromFnIterator<F, T>
    F: Callable<(), Option<T>>

    source

    Wraps a function in an iterator.

    Example

    use std::iter::from_fn;
use std::random::{next_float, thread_rng};

let sqrt2 = from_fn(|| -> Option<f64> {
    Option::some(thread_rng().next_float())
})
    .map(|v: f64| -> f64 { v * 10.0 })
    .filter(|v: f64| -> bool { v * v <= 2.0 })
    .take(100000)
    .max()
    .unwrap();

println!("{}", sqrt2); // Prints approximately 1.4142135623730951

  • fn repeat<T>(value: T) -> RepeatIterator<T>

    source

    Returns an iterator that repeats the same value indefinitely.

    Example

    let range = std::iter::repeat(42);

assert_eq!(range.next(), Option::some(42));
assert_eq!(range.next(), Option::some(42));
assert_eq!(range.next(), Option::some(42));
// ...

  • fn once<T>(value: T) -> OnceIterator<T>

    source

    Returns an iterator that repeats a single element exactly once.

    Example

    let iter = std::iter::once(42);

assert_eq!(iter.next(), Option::some(42));
assert_eq!(iter.next(), Option::none());
